## Engine and Session Factory (Lines 35–46)

```python
engine = create_async_engine(
    settings.database_url,
    echo=False,
    future=True,
    connect_args={"check_same_thread": False} if "sqlite" in settings.database_url else {},
)

AsyncSessionLocal = sessionmaker(
    engine,
    class_=AsyncSession,
    expire_on_commit=False,
)
```

- **`create_async_engine`**: Connects to the database. Uses the URL from `config.py`.
- **`"check_same_thread": False`**: Necessary for SQLite in async environments (like FastAPI) because queries can hop between OS threads.
- **`sessionmaker`**: Factory for creating async database sessions. `expire_on_commit=False` prevents SQLAlchemy from lazily dropping data from objects after a commit, which is crucial in async code where implicit IO calls are blocked.

## SQLAlchemy Models (Lines 56–109)

```python
class AgentRun(Base):
    __tablename__ = "agent_runs"

    id            = Column(Integer, primary_key=True, index=True)
    agent_name    = Column(String(64), nullable=False, index=True)
    ...
    metadata_json = Column(JSON, default={})
    
    evaluation    = relationship("EvalRecord", back_populates="agent_run", uselist=False)


class EvalRecord(Base):
    __tablename__ = "eval_records"

    id                   = Column(Integer, primary_key=True, index=True)
    agent_run_id         = Column(Integer, ForeignKey("agent_runs.id"), nullable=False)
    accuracy             = Column(Float, default=0.0)
    ...
    agent_run            = relationship("AgentRun", back_populates="evaluation")
```

- Models map Python classes to database tables.
- **`AgentRun`**: Stores the core invocation details (inputs, outputs, latency).
- **`EvalRecord`**: Stores the LLM-generated evaluation scores.
- **`ForeignKey` and `relationship`**: Establishes a 1-to-1 link between a run and its evaluation. `uselist=False` enforces that it's a single object, not a list.

## Database Lifecycle Hooks (Lines 113–122)

```python
async def init_db() -> None:
    """Create all tables if they don't exist."""
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)

async def get_session() -> AsyncSession:
    """FastAPI dependency: yields an async DB session."""
    async with AsyncSessionLocal() as session:
        yield session
```

- **`init_db`**: Executed on app startup. `run_sync` bridges the async engine to SQLAlchemy's synchronous schema creation methods.
- **`get_session`**: Used as a FastAPI `Depends()`. The `yield` acts as a context manager: it creates a session, pauses, lets the FastAPI route handler run, and then cleans up the session after the request finishes.

## Repository Functions (Lines 127–165)

```python
async def save_run(session: AsyncSession, output) -> AgentRun:
    """Persist an AgentOutput to the database."""
    run = AgentRun(
        agent_name=output.agent_name,
        ...
        task=output.task[:1000],
        result=output.result[:10_000],
    )
    session.add(run)
    await session.commit()
    await session.refresh(run)
    return run
```

- Helper to transform the agent's `AgentOutput` dataclass into an `AgentRun` SQLAlchemy model and save it.
- **String truncation `[:10_000]`**: Defensive programming to ensure gigantic texts don't blow up the database.
- **`await session.refresh(run)`**: Updates the python object with data generated by the database (specifically, the auto-incremented primary key `id`).
